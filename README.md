# Drew Peterson — Personal Landing Page

## Features

- Sticky navigation bar with smooth-scroll anchor links to each section
- Hero section with circular-cropped professional headshot, full name, and tagline
- About section summarizing academic background, majors, and career focus
- Skills section displaying technical proficiencies as navy pill badges
- Projects section with a responsive two-column card grid
- Experience section highlighting two internship roles with dates and descriptions
- Contact section with labeled badge links to LinkedIn, GitHub, and email
- All external links open in a new tab with `rel="noopener noreferrer"` for security
- Fully responsive layout from 320px screen width and wider
- No horizontal scrolling on any viewport
- Semantic HTML5 elements throughout (`<header>`, `<main>`, `<section>`, `<article>`, `<footer>`, `<nav>`)
- WCAG 2.2 Level AA accessibility compliance
- Logical heading hierarchy with no skipped levels
- Descriptive `alt` attributes on all images
- Keyboard-navigable interactive elements
- Color contrast ratios meeting WCAG minimum thresholds
- Vanilla CSS with no frameworks or dependencies
- CSS custom properties for the full color palette
- Inter typeface (Google Fonts) for headings and body
- Maximum content width of 800px, centered on the page
- Navy, white, and light-gray color scheme appropriate for a finance and CRE audience
- Clean, minimal layout with generous whitespace inspired by read.cv
- Section hierarchy inspired by brittanychiang.com (light mode)
- No JavaScript required for any core functionality
- Zero npm dependencies — no build step needed
- Single `index.html` file in the project root
- External stylesheet in a dedicated `css/` folder
- Images stored in a dedicated `images/` subfolder with relative paths
- Compatible with Chrome, Safari, and Firefox
- Mobile-responsive at 375px and wider
- Deployable as a static site with no server-side code
- Google Fonts loaded via `<link>` tag for optimized render performance
- Nav link hover and focus states with accent color transition
- Contact links styled as bordered badge buttons with hover fill
- Skill tags use consistent pill shape with navy background and white text
- Projects grid switches from single column (mobile) to two columns (640px+)
- Professional, recruiter-appropriate tone throughout all copy
- First-person, direct writing style — no buzzwords
- Page `<title>` is descriptive for SEO and accessibility
- Footer with copyright line
- No resume download links (intentional per PRD scope)
- No contact form (intentional per PRD scope)
- No animations or interactive effects (intentional per PRD scope)
- No analytics or visitor tracking embedded
- No stock photos or clip art — professional headshot only

## Contributing

This is a personal project and is not open for general contributions. However, if you would like to fork it and adapt it for your own landing page, you are welcome to do so under the terms of the MIT License.

1. Fork the repository on GitHub.
2. Clone your fork to your local machine: `git clone https://github.com/your-username/your-repo-name.git`
3. Create a new branch for your changes: `git checkout -b your-feature-name`
4. Make your edits to `index.html`, `css/stylesheet.css`, or `images/`.
5. Commit your changes with a descriptive message: `git commit -m "describe what you changed"`
6. Push your branch to GitHub: `git push origin your-feature-name`
7. Open a pull request from your branch to the main branch of your fork.
